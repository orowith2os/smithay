Integration of an x11rb X11 connection with calloop.

This is a thin wrapper around Channel. It works by spawning an extra thread reads events from the X11 connection and then sends them across the channel.

See 1 for why this extra thread is necessary. The single-thread solution proposed on that page does not work with calloop, since it requires checking something on every main loop iteration. Calloop only allows “when an FD becomes readable”.

Create a new X11 source.

The returned instance will use SendRequest to cause a ClientMessageEvent to be sent to the given window with the given type. The expectation is that this is a window that was created by us. Thus, the event reading thread will wake up and check an internal exit flag, then exit.
